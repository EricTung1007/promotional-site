# 評量報告

> 從整體表現到每一句對話，獲得完整且具體的專業回饋

## 概覽

每次練習結束後，AI Coach 會自動產生詳細的評量報告。報告涵蓋三個層次的分析：

1. **階段評量** — 各銷售階段的達成與否及改善建議
2. **綜合評核** — 整體表現的總結與學習方向
3. **逐句評價** — 每則訊息的細緻分析

這三種報告各自獨立產生，使用不同的 AI Agent，分析不同面向的表現。

---

## 報告產生流程

### 觸發時機

當您點擊「結束對話」（或客戶發起結束後您確認結束）時，系統會：

```text
檢查對話狀態 → 關閉對話 → 同時觸發三個背景任務
                              ├─ 階段評量產生
                              ├─ 綜合評核產生
                              └─ 對話摘要產生
```

> 注意：**逐句評價**不在結束時才產生，而是在對話過程中每輪訊息結束後即在背景產生（詳見下方說明）。

### 前端顯示狀態

產生過程中會顯示狀態覆蓋層：

| 狀態 | 顯示訊息 |
| --- | --- |
| 檢查中 | 正在檢查對話狀態... |
| 關閉對話 | 正在關閉對話，準備生成評核報告... |
| 產生中 | 評核報告生成中... / 我們正在為您生成詳細的評核報告 |

> 報告產生通常需要 30 秒至 2 分鐘。如超過 5 分鐘仍未完成，可嘗試重新整理頁面。

### 產生狀態

報告中的各個區塊（階段評量、綜合評核等）各自獨立產生，可能有以下狀態：

| 狀態 | 說明 |
| --- | --- |
| 排隊中 (queued) | 任務已建立，等待執行 |
| 處理中 (processing) | 正在產生內容 |
| 完成 (available) | 內容已產生完成，可以檢視 |
| 未請求 (not_requested) | 尚未觸發產生 |
| 失敗 (failed) | 產生過程發生錯誤 |
| 跳過 (skipped) | 因無對話內容或其他原因被跳過 |

## 報告頁面

報告頁面頂部顯示標題「業務員對話練習綜合評核」，並提供兩個檢視模式切換：

| 模式 | 說明 |
| --- | --- |
| 綜合評核 | 查看整體階段評量與綜合分析（預設） |
| 逐句評價 | 查看每則訊息的詳細評價 |

### 返回操作

頁面左上方提供「返回列表」按鈕，可回到儀表板或練習歷史頁面。

## 一、階段評量

### 階段評量的分析目的

階段評量是在對話結束後，由 AI 一次性回顧完整對話內容，根據管理員定義的各階段標準，判定您最終達成了哪些銷售階段，並為每個已達成的階段撰寫獨立的評量分析。

### 階段評量的資料輸入

AI 在產生階段評量時，會取得以下資訊作為分析依據：

| 輸入資料 | 說明 |
| --- | --- |
| 完整對話紀錄 | 包含每一輪的：使用者訊息、客戶內心活動、客戶回應 |
| 階段定義 | 管理員設定的所有階段，包含階段名稱與客戶狀態描述 |
| 最大階段編號 | 目前定義的最高階段數（例如 5） |
| 客戶角色資料 | 客戶的個人屬性（年齡、職業、收入、MBTI 等） |
| 客戶詳細描述 | 系統產生的客戶角色完整描述文字 |

### 階段評量的產生邏輯

```text
對話結束
  ↓
系統建立「階段評量」背景任務（狀態：排隊中）
  ↓
背景任務開始執行（狀態：處理中）
  ↓
AI Agent 接收完整對話紀錄與所有階段定義
  ↓
AI 以單次分析判定每個階段的達成情況
  ↓
輸出結構化的 JSON 格式（每個階段一筆評量）
  ↓
系統儲存評量結果，更新該次練習的「達成階段」欄位（狀態：完成）
```

**重要特性**：

- 使用**單一 AI Agent** 一次分析所有階段，而非每個階段各跑一次
- AI 可存取對話中的**客戶內心活動**（即客戶的心理分析），用以更精準地判斷階段推進
- 若對話沒有任何內容，系統會跳過評量產生（狀態標記為「跳過」）

### 階段評量的輸出結果

AI 會為每個已達成的階段產生一筆評量紀錄，每筆包含：

| 輸出欄位 | 說明 |
| --- | --- |
| 階段編號 | 所評量的階段編號 |
| 評量內容 | AI 對使用者在該階段表現的詳細分析文字 |

此外，系統會根據評量結果更新：

- **達成階段**：該次練習的最高達成階段編號（記錄於練習紀錄中）
- **是否達成所有階段**：若 AI 判定通過所有階段，會標記完成旗標

### 階段評量的評核要點

AI 在評量每個階段時，主要分析以下面向：

1. **階段目標達成度** — 對照該階段的定義描述，判斷使用者是否達成該階段的核心目標
2. **客戶狀態推進** — 對照「客戶狀態描述」，判斷客戶的心理狀態是否從該階段的起始狀態推進到下一階段
3. **對話技巧運用** — 使用者在該階段使用的溝通策略是否有效
4. **客戶互動品質** — 客戶的參與度、信任度、開放程度是否有提升
5. **未晉升原因**（若適用）— 具體分析為何未能推進至更高階段，提供改善方向

### 階段評量的頁面呈現

依據評量結果，會呈現以下其中一種情況：

**情況一：達成部分階段**

每個已達成的階段會以可展開/收合的卡片顯示：

- 標題格式：「第 N 階段：{階段名稱}」
- 內容：AI 對您在該階段表現的詳細分析

接著顯示「未晉升原因」區塊：

- 內容：AI 分析您未能晉升至下一階段的具體原因與改善建議

**情況二：達成所有階段**

- 所有階段的評量卡片
- 額外顯示：「您已達成所有階段」與恭喜訊息

**情況三：未達成任何階段**

- 顯示：「未達成任何評核階段。」
- 表示 AI 判斷對話尚未達到第一個階段的標準（可能因對話輪數不足或內容未切入銷售主題）

> 標題旁有說明按鈕，點擊可查看管理員定義的各階段定義說明。

## 二、綜合評核

### 綜合評核的分析目的

綜合評核是對整次練習的整體表現進行全面分析，提供總結性的評價與學習方向建議。與階段評量不同，綜合評核不聚焦於單一階段，而是從整體對話品質、銷售技巧、客戶經營等角度給出綜合回饋。

### 綜合評核的資料輸入

AI 在產生綜合評核時，會取得以下資訊作為分析依據：

| 輸入資料 | 說明 |
| --- | --- |
| 完整對話紀錄 | 包含每一輪的：使用者訊息、客戶內心活動、客戶回應 |
| 客戶角色資料 | 客戶的個人屬性（年齡、職業、收入、MBTI 等） |
| 客戶詳細描述 | 系統產生的客戶角色完整描述文字 |
| 練習時長 | 本次對話的總時長（秒） |

> 注意：綜合評核**不會**直接取得階段評量的結果作為輸入。兩者是由不同的 AI Agent **平行且獨立**產生的。

### 綜合評核的產生邏輯

```text
對話結束
  ↓
系統建立「綜合評核」背景任務（狀態：排隊中）
  ↓
背景任務開始執行（狀態：處理中）
  ↓
AI Agent 接收完整對話紀錄與客戶資料
  ↓
AI 從整體角度分析對話表現
  ↓
輸出 Markdown 格式的評核文字
  ↓
系統儲存評核結果（狀態：完成）
```

**重要特性**：

- 與階段評量**平行產生**，互不依賴
- 若對話沒有任何內容，系統會跳過產生（狀態標記為「跳過」）
- 產生失敗不影響階段評量與逐句評價的產生

### 綜合評核的輸出結果

綜合評核產生一段 Markdown 格式的文字，通常包含以下區塊：

| 輸出區塊 | 說明 |
| --- | --- |
| 整體表現總評 | 一段總結性描述，概述本次練習的整體表現 |
| 優點回顧 | 列舉使用者在對話中表現良好的具體面向 |
| 待改善項目 | 指出對話中可以做得更好的部分 |
| 具體改善建議 | 針對待改善項目提供可操作的建議 |
| 學習方向建議 | 建議下一步可以加強的練習方向 |

> 實際內容的區塊結構與詳細程度由 AI Agent 的 Prompt 設定決定，管理員可透過 [Prompt 管理](admin-prompts.md) 調整。

### 綜合評核的評核要點

AI 在產生綜合評核時，主要分析以下面向：

1. **對話品質** — 開場是否自然、對話節奏是否流暢、結束是否得體
2. **銷售技巧** — 是否運用開放式問題、積極傾聽、需求引導等技巧
3. **客戶關係經營** — 是否展現同理心、建立信任、維護良好互動
4. **專業知識展現** — 是否展示對產品與市場的專業理解
5. **客戶理解程度** — 是否充分了解客戶的需求、擔憂與期望
6. **整體策略** — 對話的推進策略是否合理有效

### 綜合評核的頁面呈現

- 標題：「綜合評核」
- 內容：以 Markdown 格式呈現，支援標題、列表、粗體等排版
- 若未通過任何階段，會顯示「未通過任何階段。」

## 三、逐句評價

### 逐句評價的分析目的

逐句評價是針對對話中的**每一輪訊息**進行個別分析，讓您可以精確了解每次發言的優劣之處。這是最細緻的回饋層級，適合深入檢視特定對話環節。

### 逐句評價的資料輸入

AI 在產生每一輪的逐句評價時，會取得以下資訊作為分析依據：

| 輸入資料 | 說明 |
| --- | --- |
| 截至該輪的完整對話紀錄 | 包含先前所有輪次的對話與已產生的評價 |
| 該輪的使用者訊息 | 使用者（業務員）在該輪發送的訊息 |
| 該輪的客戶內心活動 | AI 客戶在該輪的心理分析 |
| 該輪的客戶回應 | AI 客戶在該輪的實際回覆 |
| 階段定義 | 管理員設定的所有階段與最大階段編號 |
| 客戶角色資料 | 客戶的個人屬性與詳細描述 |

### 逐句評價的產生邏輯

與階段評量和綜合評核不同，逐句評價是**在對話過程中即時產生**的，而非等到對話結束後：

```text
使用者發送訊息
  ↓
系統產生 AI 客戶的內心活動與回應（同步，使用者等待）
  ↓
將回應即時回傳給使用者
  ↓
同時在背景建立「逐句評價」任務（狀態：排隊中）
  ↓
背景任務開始執行（狀態：處理中）
  ↓
AI Agent 分析該輪對話表現
  ↓
儲存評價結果（狀態：完成）
```

**重要特性**：

- **不阻塞對話流程**：使用者發送訊息後，會立即收到 AI 客戶的回應，逐句評價在背景非同步產生
- **每輪獨立產生**：每一輪對話（chat_order）對應一筆獨立的逐句評價
- **累積上下文**：後續輪次的評價可參考先前的對話紀錄與已產生的評價
- 在對話結束進入報告頁面時，大部分逐句評價通常已經產生完成

### 逐句評價的輸出結果

每一輪對話的逐句評價包含：

| 輸出欄位 | 說明 |
| --- | --- |
| 對話輪次 (chat_order) | 該評價對應的第幾輪對話 |
| 評價內容 | AI 對該輪使用者表現的詳細分析（Markdown 格式） |
| 產生狀態 | 該輪評價的當前狀態（完成 / 處理中 / 失敗等） |

### 逐句評價的評核要點

AI 在產生逐句評價時，主要分析以下面向：

1. **表達方式** — 用字遣詞是否得當、語氣是否合適、是否清晰易懂
2. **銷售技巧** — 該輪是否有效運用提問、傾聽、引導等銷售技巧
3. **客戶關係維護** — 該輪回應是否有助於建立或維持與客戶的信任關係
4. **階段推進效果** — 該輪對話對推進銷售階段是否有正面貢獻
5. **客戶心理回應** — 分析客戶在該輪的內心反應，幫助理解對話的實際效果
6. **具體改善建議** — 針對該輪的具體情境提出可操作的改進方向

### 逐句評價的頁面呈現

切換至「逐句評價」模式後，每一輪對話（您的訊息 + AI 客戶回覆）會顯示為一張卡片，包含：

| 區塊 | 說明 |
| --- | --- |
| 您的訊息 | 右側藍色氣泡，顯示您發送的訊息內容 |
| AI 客戶回覆 | 左側灰色氣泡，顯示 AI 客戶的回覆 |
| AI 內心思考 | 標題「AI內心思考」，顯示 AI 分析客戶在該輪對話中的心理活動與想法 |
| 逐句評價 | 標題「逐句評價」，Markdown 格式的詳細分析 |

每一輪對話下方都有獨立的評分區塊，評分方式與綜合評核的回饋評分相同（正面 / 中立 / 負面）。

## 三種報告的比較

| 面向 | 階段評量 | 綜合評核 | 逐句評價 |
| --- | --- | --- | --- |
| **分析範圍** | 各銷售階段的達成情況 | 整次練習的整體表現 | 每一輪訊息的個別表現 |
| **產生時機** | 對話結束後 | 對話結束後 | 對話過程中（每輪即時背景產生） |
| **AI Agent** | stage_review_agent | final_review_agent | overall_assessment_agent |
| **主要輸入** | 完整對話 + 階段定義 | 完整對話 + 練習時長 | 截至該輪的對話 + 階段定義 |
| **輸出格式** | 每階段一筆結構化評量 | 一段 Markdown 綜合分析 | 每輪一筆 Markdown 評價 |
| **互相依賴** | 獨立產生 | 獨立產生 | 獨立產生 |
| **適合用途** | 了解階段達成與未晉升原因 | 獲得整體回饋與學習方向 | 逐一檢視每次發言的優劣 |

## 回饋評分

### 綜合評核的回饋

在綜合評核模式下方，提供回饋評分區塊：

- 標題：「喜歡這次評核內容嗎？」
- 說明：「針對階段評核與綜合評核提供意見回饋，以協助提升服務品質與整體體驗。」

| 選項 | 操作 | 說明 |
| --- | --- | --- |
| 正面 | 點擊讚按鈕 | 覺得評核內容有幫助，可附加留言（選填） |
| 中立 | 預設狀態（不需操作） | 預設值，無特別意見。再次點擊已選的按鈕可回到中立 |
| 負面 | 點擊倒讚按鈕 | 覺得評核需要改善，需附加留言說明原因（必填） |

### 逐句評價的回饋

每一輪對話的逐句評價下方都有獨立的評分區塊，評分方式相同（正面 / 中立 / 負面）。這讓您可以針對 AI 在特定訊息上的評價品質提供回饋，幫助系統持續優化。

## 評量報告的使用建議

### 如何有效利用評量報告

1. **先看階段評量** — 了解自己在哪些階段表現良好、哪些需要加強
2. **閱讀未晉升原因** — 如果未達成所有階段，仔細閱讀原因分析
3. **查看綜合評核** — 了解 AI 對整體表現的觀察與建議
4. **切換逐句評價** — 針對表現不佳的階段，回顧具體的對話細節
5. **關注 AI 內心思考** — 了解客戶在各階段的心理狀態，有助下次調整策略
6. **提供回饋評分** — 您的評分有助於系統持續優化評量品質

### 持續改善循環

```text
查看評量 → 找出弱點 → 重新挑戰相同客戶 → 對比前後表現 → 逐步提升
```
