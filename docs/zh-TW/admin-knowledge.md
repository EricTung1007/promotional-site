# 知識庫管理

> 上傳與管理文件資料，為知識助手提供精準的知識來源

## 概覽

知識庫管理功能讓管理員上傳、查看和刪除知識庫中的文件資料。這些資料會經過向量化處理，成為知識助手（RAG）的檢索來源，協助使用者在練習過程中查詢相關的銷售知識與產品資訊。

> 此功能僅對具有知識庫管理權限的管理員開放。
>
> **注意**：目前知識庫管理主要透過後端 API 操作，前端管理介面尚在開發中。以下為功能說明，實際操作方式請依公司內部流程為準。

---

## 文件上傳

### 支援的檔案格式

| 格式 | 副檔名 | 說明 |
| --- | --- | --- |
| PDF | .pdf | 包含文字、表格、圖片的完整文件 |
| Word 文件 | .docx | Microsoft Word 文件 |
| 純文字 | .txt | 純文字格式 |
| HTML | .html | 網頁格式文件 |
| Markdown | .md | Markdown 格式文件 |

### 上傳流程

1. 進入文件上傳頁面
2. 選擇要上傳的檔案
3. 系統自動執行以下處理：

```text
文件解析 → 內容分塊 → 向量化嵌入 → 建立索引
```

### 自動處理說明

| 步驟 | 說明 |
| --- | --- |
| 文件解析 | 解析文件內容，識別文字、表格與圖片 |
| 內容分塊 | 將文件內容拆分為適合檢索的小段落（Chunks） |
| 向量化嵌入 | 將每個段落轉換為向量表示，用於語義搜尋 |
| 建立索引 | 將向量資料存入資料庫，供知識助手檢索使用 |

### 分塊類型

上傳後的文件會被分為以下三種類型的 Chunk：

| 類型 | 說明 |
| --- | --- |
| TEXT | 文字段落內容 |
| TABLE | 表格結構資料 |
| IMAGE | 圖片內容（以 Base64 編碼儲存） |

## 查看知識庫

### 文件列表

管理員可查看已上傳至知識庫的所有文件分塊，每筆紀錄包含：

| 欄位 | 說明 |
| --- | --- |
| 文件名稱 | 原始上傳的文件名稱 |
| 公司 | 所屬公司（用於多租戶隔離） |
| 部門 | 所屬部門 |
| 分塊類型 | TEXT / TABLE / IMAGE |
| 起始頁碼 | 此分塊在原始文件中的起始頁 |
| 結束頁碼 | 此分塊在原始文件中的結束頁 |
| 更新時間 | 最後更新的時間 |
| 建立者 | 上傳此文件的使用者 |
| 摘要 | 此分塊的文字摘要 |
| 原始文字 | 此分塊的完整原始內容（可選擇是否顯示） |

### 篩選條件

| 條件 | 說明 |
| --- | --- |
| 日期範圍 | 依更新時間篩選（起始日期 ~ 結束日期） |
| 公司 | 篩選特定公司的資料 |
| 部門 | 篩選特定部門的資料 |
| 文件名稱 | 依文件名稱搜尋 |
| 分塊類型 | 篩選特定類型（TEXT / TABLE / IMAGE） |

### 聚合分析

提供知識庫的統計概覽：

| 指標 | 說明 |
| --- | --- |
| 總分塊數 | 知識庫中所有分塊的總數量 |
| 總文件數 | 已上傳的不重複文件數量 |

支援依以下維度分組統計：

- 依公司分組
- 依部門分組
- 依文件名稱分組
- 依分塊類型分組

## 刪除文件

管理員可批次刪除知識庫中的文件分塊。

### 刪除操作

1. 使用篩選條件選定要刪除的分塊範圍
2. 執行刪除操作
3. 系統回報刪除結果：
   - 已刪除筆數
   - 剩餘版本數
   - 刪除時間

> 刪除操作不可逆轉。刪除後，知識助手將無法再檢索到這些內容。

---

## 多租戶隔離

知識庫採用嚴格的多租戶資料隔離機制：

- 每份上傳的文件會自動關聯上傳者所屬的公司
- 使用者只能透過知識助手查詢到自己公司的知識庫內容
- 管理員只能管理自己權限範圍內的知識庫資料
- 不同公司之間的知識庫完全隔離，互不影響

## 知識庫管理建議

### 內容規劃

| 建議上傳的內容 | 說明 |
| --- | --- |
| 產品資料 | 保險產品的詳細規格、條款、費率表 |
| 銷售話術 | 經過驗證的銷售話術與應對技巧 |
| 常見問題集 | 客戶常見問題與標準回答 |
| 公司規範 | 內部流程、合規要求、理賠規範 |
| 市場資訊 | 市場趨勢、競品比較、客戶案例 |

### 維護建議

1. **定期更新** — 產品改版或政策變更時，及時更新對應文件
2. **清理過期資料** — 定期刪除已過期或不再適用的內容
3. **格式優化** — 上傳前整理文件格式，確保內容結構清晰
4. **分類管理** — 使用有意義的檔案名稱，便於後續管理
